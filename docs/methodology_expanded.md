## 3. Problem Formulation

### 3.1 Setup

Consider a source asset $S$ and targets $\{T_1, \ldots, T_N\}$. For each asset $A$, we observe features $\mathbf{x}_t^A \in \mathbb{R}^d$ at time $t$, a prediction $p_t^A \in [0,1]$ generated by a machine learning model, and the realized return $r_{t+1}^A$. The features capture market state information including price momentum, volatility regimes, volume patterns, and cross-asset correlations.

The prediction $p_t^A$ represents the model's estimate of the probability that asset $A$ will experience a positive return over the next period. This prediction is generated by a gradient boosted tree (GBT) model trained using walk-forward validation on historical data. The specific features and model architecture are detailed in Section 6.

Our setup assumes that predictions are generated independently for each asset, without any cross-asset information sharing. This isolates the decision-layer transfer problem from the prediction-layer transfer problem, allowing us to study the former in a controlled setting.

### 3.2 Decision policy

A decision policy $\pi$ maps the prediction and contextual information to a trading signal: $s_t = \mathbb{1}[p_t \geq \tau(\mathbf{c}_t)]$, where $\tau(\mathbf{c}_t) \in \mathcal{T} = \{\tau_1, \ldots, \tau_K\}$ is a context-dependent threshold selected from a discrete set. The trading signal $s_t \in \{0, 1\}$ indicates whether to take a long position in the asset.

The context vector $\mathbf{c}_t \in \mathbb{R}^3$ captures three market regime indicators:

1. **Empirical Mode Decomposition (EMD) trend**: The first intrinsic mode function of the price series, capturing short-term momentum.

2. **Volatility quantile**: The current volatility level relative to its historical distribution, indicating whether the market is in a high or low volatility regime.

3. **Trend strength**: A measure of the consistency of recent price movements, distinguishing trending from mean-reverting markets.

These three features were selected to capture the key dimensions of market state that are likely to affect optimal threshold selection. The discrete threshold set $\mathcal{T} = \{0.35, 0.45, 0.55, 0.65, 0.75, 1.00\}$ spans a range from aggressive (low threshold, high trade frequency) to conservative (high threshold, low trade frequency).

### 3.3 Contextual bandit formulation

We formulate the threshold selection problem as a contextual bandit. At each time $t$:

1. **Observe context**: The agent observes the context vector $\mathbf{c}_t$ representing the current market state.

2. **Select action**: The agent selects an action $a_t \in \{1, \ldots, K\}$ corresponding to one of the $K$ thresholds in $\mathcal{T}$.

3. **Receive reward**: The agent receives a reward $r_t$ based on the trading performance when using the selected threshold. Specifically, the reward is the excess return relative to a benchmark (e.g., no trading or a fixed threshold).

The objective is to maximize cumulative reward $\sum_{t=1}^T r_t$ over the evaluation period. We use the LinUCB algorithm (Li et al., 2010) with disjoint linear models, where the expected reward for action $k$ is modeled as $\mathbb{E}[r_t | a_t = k, \mathbf{c}_t] = \boldsymbol{\theta}_k^\top \mathbf{c}_t$.

The LinUCB algorithm balances exploration and exploitation by selecting actions according to:

$$a_t = \arg\max_k \left( \boldsymbol{\theta}_k^\top \mathbf{c}_t + \alpha \sqrt{\mathbf{c}_t^\top \mathbf{A}_k^{-1} \mathbf{c}_t} \right)$$

where $\mathbf{A}_k = \mathbf{I} + \sum_{\tau: a_\tau = k} \mathbf{c}_\tau \mathbf{c}_\tau^\top$ is the design matrix for action $k$, and $\alpha > 0$ is an exploration parameter. We set $\alpha = 0.3$ based on preliminary experiments.

### 3.4 Zero-shot transfer

In the zero-shot transfer setting, a policy $\pi_S$ trained on source asset $S$ is deployed on target asset $T$ without any target-side learning. This is the most challenging transfer scenario, as the policy must generalize from the source context distribution to the target context distribution without adaptation.

Zero-shot transfer is attractive in practice because it eliminates the need for target-side data collection and retraining. However, it also carries the risk that the source policy may not generalize to the target asset's market dynamics. Our work identifies the specific conditions under which this generalization fails.

Formally, the transferred policy on target $T$ is:

$$\pi_T(\mathbf{c}_t, p_t) = \mathbb{1}[p_t \geq \tau_{a_t^*}] \quad \text{where} \quad a_t^* = \arg\max_k \left( \boldsymbol{\theta}_k^S \mathbf{c}_t + \alpha \sqrt{\mathbf{c}_t^\top (\mathbf{A}_k^S)^{-1} \mathbf{c}_t} \right)$$

where $\boldsymbol{\theta}_k^S$ and $\mathbf{A}_k^S$ are the parameters learned on the source asset.

### 3.5 The collapse question

We now formalize the collapse phenomenon that motivates our work.

**Definition 1** (Severe collapse). A policy $\pi_S$ exhibits *severe collapse* on target $T$ if:

1. **Modal-action share**: $\max_k \hat{p}_k \geq 0.90$, where $\hat{p}_k = \frac{1}{T}\sum_{t=1}^T \mathbb{1}[a_t = k]$ is the empirical frequency of action $k$.

2. **Normalized entropy**: $H(\hat{\mathbf{p}})/\log K \leq 0.25$, where $H(\hat{\mathbf{p}}) = -\sum_k \hat{p}_k \log \hat{p}_k$ is the entropy of the action distribution.

Severe collapse indicates that the policy has effectively degenerated into a static rule, selecting the same action for almost all contexts. This defeats the purpose of using a contextual bandit, which is to adapt the threshold based on market conditions.

**Definition 2** (Source degeneracy). A source policy $\pi_S$ is *degenerate* if its modal-action share during source-side evaluation exceeds a threshold $\gamma_S$. We use $\gamma_S = 0.90$ throughout this paper.

Source degeneracy is a necessary condition for severe target collapse. If the source policy already concentrates on a single action, it cannot adapt to different contexts on the target asset. However, source degeneracy is not sufficient for target collapse—the target context distribution must also be compatible with the degenerate policy.

**Definition 3** (Behavioral equivalence). A policy $\pi_S$ is *behaviorally equivalent* to a static threshold $\tau_k$ on target $T$ if:

$$\pi_S(\mathbf{c}_t, p_t) = \mathbb{1}[p_t \geq \tau_k] \quad \forall \mathbf{c}_t \in \mathcal{C}_T$$

where $\mathcal{C}_T$ is the set of all possible contexts on target $T$.

Behavioral equivalence is a stronger condition than severe collapse. A policy can exhibit severe collapse without being behaviorally equivalent to any single threshold—it might occasionally select different actions for extreme contexts. However, in practice, severe collapse often implies approximate behavioral equivalence.

## 4. Algorithm: FCTT-DA

### 4.1 Architecture

The Frozen Contextual Threshold Transfer with Degeneracy Audit (FCTT-DA) framework consists of three modules designed to diagnose and mitigate cross-asset policy transfer failures.

**Module 1: Local Predictor**. For each asset, we train a gradient boosted tree (GBT) using walk-forward validation. The training window is 504 trading days (approximately two years), the test window is 63 trading days (approximately one quarter), and the step size is 63 days. This setup ensures that predictions are generated out-of-sample, mimicking real-world deployment conditions.

The GBT model uses 20 technical and fundamental features (detailed in Section 6.2) to predict the probability of a positive excess return over the next period. Hyperparameters are tuned via cross-validation within each training window.

**Module 2: Frozen Contextual Threshold Policy**. The threshold selection policy is a LinUCB algorithm with $\alpha = 0.3$ and 3D context features: EMD trend, volatility quantile, and trend strength. The policy is trained on the source asset's training period and then frozen (no further learning) for transfer to target assets.

The threshold set is $\mathcal{T} = \{0.35, 0.45, 0.55, 0.65, 0.75, 1.00\}$. The threshold $1.00$ corresponds to "NoGating"—never taking a position, which serves as a conservative default when the policy has insufficient confidence.

**Module 3: Degeneracy Audit**. Before deploying the transferred policy on a target asset, we perform pre-deployment diagnostics on the source policy. The audit includes three metrics:

- **Normalized action entropy**: $H(\hat{\mathbf{p}})/\log K$, measuring the diversity of action selections.
- **Modal-action share**: $\max_k \hat{p}_k$, measuring the concentration on the most frequent action.
- **Effective actions**: $\exp(H(\hat{\mathbf{p}}))$, an alternative measure of action diversity.

The rejection rule is: if source modal share $\geq 0.90$, reject the transfer. This threshold is conservative, ensuring that only sufficiently diverse policies are transferred.

### 4.2 Comparison baselines

To evaluate the performance of FCTT-DA, we compare against three baseline approaches:

1. **NoGating**: A static threshold of 0.50, representing the common practice of using a fixed decision rule without contextual adaptation.

2. **Fixed-$\tau$**: Static thresholds at $\tau \in \{0.35, 0.45, 0.50, 0.55, 0.65, 0.75\}$, representing a grid search over possible fixed thresholds.

3. **Behavioral-equivalent**: The fixed threshold closest to the learned policy's behavior on the source asset, representing the best possible static approximation of the contextual policy.

These baselines allow us to assess whether the contextual policy provides genuine value over simpler alternatives. If the contextual policy collapses to a static rule, it should perform no better than the behavioral-equivalent baseline.

### 4.3 Statistical inference

We employ rigorous statistical methods to ensure the robustness of our findings:

- **Stationary bootstrap**: We use the stationary bootstrap (Politis and Romano, 1994) with 2,000 replications and a mean block length of 10 periods. This method preserves the temporal dependence structure of the data while providing valid confidence intervals.

- **Paired tests**: All comparisons are performed on the same return series, ensuring that differences are not driven by different market conditions. We use paired t-tests for means and paired Wilcoxon tests for medians.

- **False discovery rate control**: We apply the Benjamini-Hochberg procedure (Benjamini and Hochberg, 1995) at a 5% false discovery rate to account for multiple testing across the 234 source-target pairs.

## 5. Theoretical Analysis

### 5.1 Source diversity predicts target collapse

We now establish the theoretical link between source diversity and target collapse. Our analysis proceeds in three steps, corresponding to three propositions.

**Proposition 1**. *Let $\pi_S$ be a $K$-action LinUCB policy trained on source $S$. Define source diversity as $D_S = \exp(H_S)/K$, where $H_S$ is the entropy of the action distribution on the source. If $D_S \leq \delta$ for some $\delta < 1$, then for any target $T$ with context distribution $P_T$:*

$$\max_k \mathbb{P}_T[a_t = k] \geq \max_k \mathbb{P}_S[a_t = k] - \epsilon \cdot W_1(P_S, P_T)$$

*where $W_1$ is the Wasserstein-1 distance between source and target context distributions, and $\epsilon$ is a Lipschitz constant depending on the policy parameters.*

**Proof sketch**: The policy's action selection is a continuous function of the context. When the source policy concentrates on a single action (low $D_S$), the target policy inherits this concentration up to an error proportional to the context distribution shift. The Lipschitz constant $\epsilon$ captures the sensitivity of the policy to context changes.

**Interpretation**: When the source policy concentrates on few actions (low $D_S$), the target policy inherits this concentration. The key empirical prediction is: source diversity is a leading indicator of target collapse. If $D_S$ is low, target collapse is likely regardless of the target context distribution.

### 5.2 Action-region geometry

**Proposition 2**. *For a $K$-action LinUCB policy with weight matrix $\Theta \in \mathbb{R}^{K \times 3}$, the decision region for action $k$ is a convex polyhedral cone in context space. If the target context cloud has diameter $d_T$ and the distance from the cloud centroid to the nearest decision boundary exceeds $d_T/2$, the policy selects the same action for all target contexts.*

**Proof sketch**: The LinUCB policy selects actions based on linear functions of the context. The decision boundaries are hyperplanes, and the regions where each action is optimal are convex polyhedra. If the target context cloud is entirely contained within one region, the policy selects the same action for all contexts.

**Testable implication**: The decision margin $m = \min_t (\max_k \theta_k^\top \mathbf{c}_t - \max_{j \neq k^*} \theta_j^\top \mathbf{c}_t)$ predicts collapse. A large margin indicates that the context cloud is far from decision boundaries, making collapse more likely.

### 5.3 Behavioral equivalence

**Proposition 3**. *If $\pi_S$ collapses to threshold $\tau_{k^*}$ on target $T$, then for any metric $M$ depending only on signals:*

$$M(\pi_S, T) = M(\tau_{k^*}, T)$$

*where $M(\tau_{k^*}, T)$ is the metric computed using the static threshold $\tau_{k^*}$.*

**Proof sketch**: If the policy selects the same action for all contexts, the resulting signals are identical to those produced by the corresponding static threshold. Since the metric depends only on signals, it must be the same.

**Implication**: Comparing against 0.50 is misleading when the policy has collapsed. The correct comparison is against the behavioral-equivalent threshold $\tau_{k^*}$. This insight motivates our experimental design, which includes behavioral-equivalent baselines.

**Corollary 1**. *If $\pi_S$ is behaviorally equivalent to $\tau_{k^*}$ on target $T$, then:*

$$\text{Sharpe}(\pi_S, T) = \text{Sharpe}(\tau_{k^*}, T)$$

*where $\text{Sharpe}$ is the Sharpe ratio of the trading strategy.*

This corollary provides a practical test for behavioral equivalence: if the contextual policy's Sharpe ratio equals that of some static threshold, the policy has likely collapsed to that threshold.
